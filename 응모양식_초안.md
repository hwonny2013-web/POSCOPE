## 4. 한 줄 요약 (200자 이내)

각 본부 담당자가 매일 수동으로 찾아보던 해외 발주처 공고·뉴스를 6시간 주기로 6개 소스에서 자동 수집하고 AI/키워드로 본부별 분류해 실시간 대시보드로 보여주며, 거래선 미팅이력과 담당자 진행상황까지 한 곳에서 관리하는 웹 플랫폼입니다.

## 5. 배경 및 문제인식 (As-Is, 2000자 이내)

해외 철강·소재바이오·에너지·가스 4개 본부는 각자 담당하는 거래선과 지역의 발주 기회(PJT/INQ)를 파악하기 위해 Google News, KOICA ODA, World Bank 조달공고, 해외건설협회 공지, 업계 전문지(ENR, Rigzone, Offshore Energy 등) 등 다수의 사이트를 담당자가 매일 직접 방문해 확인해야 했습니다.

이 과정의 구체적인 문제는 다음과 같습니다.

1) **시간 소모와 누락 위험**: 담당자 한 명이 수십 개 사이트와 키워드를 매일 수동으로 검색·정리하는 데 상당한 시간이 소요되며, 휴가·출장 등으로 하루라도 누락되면 마감이 임박한 발주 정보를 놓칠 위험이 있습니다.

2) **본부 간 분류 기준 불일치**: 어떤 발주가 철강본부 소관인지, 에너지/가스/소재바이오 소관인지 담당자 개인 판단에 의존해 분류 기준이 사람마다 달랐고, 분류 결과를 모아보는 전사 차원의 통합 뷰가 없었습니다.

3) **이력 관리의 파편화**: 발굴한 PJT 정보는 개인 엑셀 파일로 관리되어 본부장·관리자가 전체 파이프라인(검토중→오퍼제출→협상중→수주/유찰)을 실시간으로 파악하기 어려웠고, 담당자가 바뀌면 그동안의 진행 경위(누가 언제 무엇을 했는지)가 구두 인계에 의존해 소실되기 쉬웠습니다.

4) **거래선 미팅이력과 발주 정보의 단절**: Salesforce에 기록되는 거래선 미팅·활동 이력과, 뉴스/공고로 파악한 발주 기회 정보가 서로 다른 시스템에 있어 영업 담당자가 양쪽을 오가며 수동으로 대조해야 했습니다.

5) **신규/타본부 인력의 진입장벽**: 어떤 사이트를 봐야 하는지, 어떤 키워드로 검색해야 하는지가 문서화되어 있지 않아 담당자 개인의 노하우에 의존했고, 인력 교체 시 동일한 품질의 정보 수집이 재현되기 어려웠습니다.

## 6. 해결 방법 및 변화 모습 (To-Be, 2000자 이내)

POSCOPE는 발주 정보 수집부터 본부별 분류, 진행 관리, 통계 리포트까지를 하나의 웹 애플리케이션(Flask + SQLite)으로 통합했습니다.

**자동 수집·분류**: Windows 작업 스케줄러에 6시간 주기로 등록된 크롤러가 Google News RSS, KOICA ODA, World Bank API(진행 프로젝트/조달공고), 해외건설협회, 업계 전문지 RSS 등 6개 소스를 자동으로 순회합니다. 본부별·세부사업별 키워드 사전으로 1차 분류하고, OpenAI API 키가 설정되면 GPT가 발주처·품목·물량까지 추출해 정확도를 높입니다(키 없이도 키워드 기반으로 100% 동작). 링크·제목 정규화 기반 중복 제거로 같은 기사가 여러 매체에 실려도 한 건만 적재됩니다.

**실시간 통합 대시보드**: 웹 화면 하나에서 출처(자동/수동), 본부, 구분(PJT/INQ), 지역, SF 이력 여부, 진행상태로 필터링하고 검색할 수 있으며, 카드를 클릭하면 발주처·금액·품목·AI 요약·출처 링크·진행로그 타임라인·거래선 미팅이력을 한 화면에서 확인할 수 있습니다.

**본부별 현황 페이지**: 4개 본부(철강/소재바이오/에너지/가스) 각각의 파이프라인 단계별 분포, 지역별 분포, 마감 임박 PJT를 시각화한 전용 페이지를 제공해 본부장이 별도 보고 없이 직접 현황을 파악할 수 있습니다.

**진행 이력의 자산화**: 담당자가 PJT 상세 화면에서 진행 내용을 한 줄씩 남기면 시간순 타임라인으로 누적되어, 담당자가 바뀌어도 그간의 경위가 그대로 인계됩니다. 상태(검토중/오퍼제출/협상중/수주/유찰) 변경도 같은 화면에서 즉시 반영됩니다.

**거래선 이력 연동 준비**: Salesforce의 거래선 활동(Task/Event) 이력을 담당자 기준으로 조회해 PJT 상세 화면에 자동 병합하는 기능을 코드 레벨로 구현해 두었으며, 사내 Salesforce 관리자 권한 확보 후 연동 정보만 입력하면 즉시 활성화됩니다.

**비개발자도 운영 가능한 배포**: 설치(`1_설치.bat`)·서버 실행(`2_서버실행.bat`)·정기 수집 등록(`3_정기수집_등록.bat`) 모두 더블클릭 한 번으로 끝나도록 배치파일화해, IT 지식이 없는 담당자도 본인 PC에서 바로 운영할 수 있습니다. 엑셀 업로드/내보내기 기능으로 기존 보고 양식과도 호환됩니다.

## 7. 정량적 효과 (1000자 이내, 추정치)

- **자동수집 비율 97.6%**: 현재 누적 205건의 PJT/INQ 중 201건(97.6%)이 사람 손을 거치지 않고 자동 수집·분류되었으며, 수동입력은 4건(2.4%)에 불과합니다. (앱 내 통계 기준 실측치)
- **본부별 자동 분류 처리**: 에너지 90건(44%), 철강 49건(24%), 소재바이오 49건(24%), 가스 17건(8%) — 4개 본부 분량을 사람이 나눠 본 적 없이 한 번에 분류·집계.
- **수집 주기 6시간**: 기존에는 담당자가 하루 1회(영업일 기준) 수동 확인하던 것을 6시간(1일 4회) 주기 자동 점검으로 전환해, 마감 임박 공고를 놓칠 위험이 약 4배 줄어든 것으로 추정.
- **담당자 업무시간 절감(추정)**: 본부별 1~2명이 매일 30분~1시간씩 다수 사이트를 수동 검색하던 작업이 자동화되어, 4개 본부 합산 월 40시간 이상의 단순 검색·정리 업무가 절감될 것으로 추정.
- **설치~가동 시간 5분 이내**: 배치파일 기반 설치로 별도 서버 구축 없이 개인 PC에서 5분 내 가동 가능 (기존 IT 인프라 신청·승인 절차 불필요).

## 8. 정성적 효과 (2000자 이내)

**전사 가시성 확보**: 4개 본부의 발주 기회가 한 화면에 모여 비교·검색 가능해지면서, 경영진과 본부장이 별도 보고를 요청하지 않아도 실시간으로 파이프라인을 파악할 수 있게 되었습니다. 본부별 현황 페이지는 분기 보고 자료의 기초 데이터로도 바로 활용 가능합니다.

**개인 노하우의 조직 자산화**: 기존에는 "어떤 사이트를, 어떤 키워드로" 봐야 하는지가 담당자 개인의 경험에만 머물러 있었지만, 크롤러의 소스·키워드 사전이 코드와 설정 파일로 문서화되어 누구나 동일한 품질로 정보를 수집할 수 있게 되었습니다. 담당자 교체 시에도 진행로그 타임라인 덕분에 PJT별 경위가 그대로 인계되어 업무 연속성이 높아졌습니다.

**의사결정 속도 향상**: 발주처·품목·금액·진행상태가 구조화된 데이터로 쌓이면서 검색·필터링만으로 원하는 정보를 즉시 찾을 수 있어, 기존 엑셀 파일을 열어 일일이 찾던 방식보다 의사결정에 필요한 정보 탐색 시간이 크게 줄었습니다.

**일관된 분류 기준**: "철강이 필요한 발주는 구매처 업종과 무관하게 철강본부"와 같은 분류 원칙이 시스템 규칙으로 일관되게 적용되어, 본부 간 책임 소재 다툼이나 중복 보고 가능성이 줄었습니다.

**확장 가능한 구조**: 현재는 4개 본부·6개 수집 소스로 운영되지만, 키워드 사전과 소스 목록을 설정 파일에 추가하는 방식으로 다른 본부나 신규 지역 조달 포털로 손쉽게 확장할 수 있도록 설계되었습니다. Salesforce 연동도 코드는 완성되어 있어 권한만 확보되면 바로 활성화됩니다.

**비개발자 친화적 운영**: 별도 IT 인프라 신청이나 서버 승인 절차 없이 담당자 PC에서 배치파일 실행만으로 도입할 수 있어, 다른 본부·팀으로의 확산 장벽이 낮습니다.

## 9. 자유 첨언

- Salesforce 거래선 미팅이력 연동 기능은 코드 레벨로 구현이 완료되어 있으나, 사내 Salesforce Connected App 생성에 필요한 관리자 권한을 아직 확보하지 못해 현재는 비활성 상태입니다. 권한 확보 시 설정값만 입력하면 즉시 가동됩니다.
- OpenAI API 키를 등록하면 키워드 기반 분류보다 더 정교한 발주처/품목/물량 자동 추출이 가능하며, 현재는 키 없이도 키워드 규칙 기반으로 완전히 동작하도록 설계해 비용 없이도 핵심 기능을 사용할 수 있습니다.
- 코드 전체는 GitHub 저장소로 첨부하며(외부 링크 항목 참조), 추가 본부·지역으로의 확장도 설정 파일 수정만으로 가능합니다.
